# 📅 주차별 목표

## 1주차: 프로젝트 계획 및 기본 구조

### 목표

프로젝트 계획 수립, 개발 환경 구성, GitHub 데이터 가져와서 렌더링

### 주요 작업

- 프로젝트 기획 및 요구사항 분석
- 개발 환경 구성
  - Vite + React + TypeScript 프로젝트 초기화
  - Tailwind CSS 4 설정
  - ESLint & Prettier 설정
  - path alias 설정
  - pre-commit hook 설정
- Express 서버 구성
  - 기본 서버 초기화
  - CORS 설정
  - 환경변수 관리 (.env)
  - GitHub API 프록시 엔드포인트
- GitHub API 연동
  - Personal Access Token 발급
  - 레포지토리 및 PR 목록 조회 API
  - PR 상세 정보 조회 API
  - 서버에서 토큰 관리
- 기본 UI 구현
  - 레포지토리 검색 페이지
  - PR 목록 표시
  - 로딩 및 에러 상태 처리

### 산출물

- 완성된 개발 환경 설정
- Express 서버 구축
- GitHub API 프록시 서버
- GitHub 데이터 조회 및 렌더링 기능
- 레포지토리 검색 페이지

## 2주차: LLM API 연동

### 목표

에디터 페이지 구현 및 AI 초안 생성 기능

### 주요 작업

- 에디터 페이지 구현
  - 마크다운 에디터 통합
  - 제목 입력 필드
  - PR 정보 참조 패널
- AI 생성 기능 (선택적)
  - LLM API 키 설정 (.env)
  - 프롬프트 작성
  - Express에 LLM API 엔드포인트 추가
  - "AI로 초안 생성" 버튼 구현
  - PR 본문 및 커밋 로그 기반 콘텐츠 생성
  - 생성된 콘텐츠를 에디터에 자동 입력

### 산출물

- 완성된 에디터 페이지
- AI 기반 콘텐츠 자동 생성 기능

## 3주차: 저장 및 조회 기능

### 목표

생성된 글 저장 및 불러오기

### 주요 작업

- 로컬스토리지 서비스 구현
  - 포스팅 CRUD 기능
  - 데이터 타입 정의
- 메인 페이지 구현
  - 포스팅 목록 그리드 레이아웃
  - 포스팅 카드 컴포넌트
  - 빈 상태 처리
- 포스팅 상세 페이지 구현
  - 마크다운 렌더링
  - 메타 정보 표시
  - 수정/삭제 기능
- 전체 플로우 연결
  - 작성 → 저장 → 조회 → 수정 → 삭제
- 최종 테스트 및 버그 수정

### 산출물

- 완성된 메인 페이지
- 포스팅 상세 페이지
- 전체 기능 통합 및 테스트 완료
