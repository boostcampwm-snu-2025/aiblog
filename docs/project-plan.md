# AI 기술 블로그 프로젝트 기획서

## 📋 프로젝트 개요

AI가 GitHub PR과 커밋 로그 참고하여 포스팅을 자동으로 작성해주는 개발 블로그

---

## 🎯 요구사항 분석

### 1. 메인 페이지 (/)

#### 1.1 포스팅 목록 노출

- 네이버 블로그의 프롤로그 형태로 포스팅을 그리드 레이아웃으로 표시한다
- 각 포스팅 카드에는 제목과 본문 미리보기(몇 줄)가 표시된다
- 포스팅 카드 클릭 시 상세 페이지로 이동한다

#### 1.2 글쓰기 진입

- 글쓰기 버튼을 통해 레포지토리 검색 페이지로 이동한다

### 2. 포스팅 상세 페이지 (/post/:id)

#### 2.1 포스팅 정보 표시

- 제목, 작성/수정 날짜를 표시한다
- 연결된 GitHub 레포지토리 링크를 표시한다 (연동된 경우)
- 연결된 PR 링크를 표시한다 (PR 기반 포스팅인 경우)
- 마크다운으로 작성된 본문 내용을 렌더링하여 표시한다

#### 2.2 포스팅 관리

- 수정 버튼을 통해 에디터 페이지로 이동하여 내용을 수정할 수 있다
- 삭제 버튼을 통해 포스팅을 삭제할 수 있다
- 삭제 시 확인 모달을 표시한다

### 3. 레포지토리 검색 페이지 (/post/create/search)

#### 3.1 작성 방식 선택

- "그냥 작성하기" 버튼으로 PR 연동 없이 빈 에디터로 바로 진입할 수 있다
- "PR로 작성하기" 영역에서 레포지토리 검색이 가능하다

#### 3.2 레포지토리 검색 (PR로 작성하기 선택 시)

- 레포지토리 이름(owner/repo 형식)을 입력하고 검색 버튼을 누른다
- 검색 결과로 해당 레포지토리의 PR 목록이 표시된다

#### 3.3 PR 목록 표시

- PR 제목, PR 번호, 생성 날짜가 표시된다
- PR 상태(open/closed/merged)가 표시된다
- PR을 선택할 수 있는 버튼이 있다

#### 3.4 글쓰기 진입

- PR을 선택하고 "이 PR로 글쓰기" 버튼을 누르면 에디터 페이지로 이동한다
- 이동 시 쿼리 파라미터에 선택한 PR 정보를 전달한다
  - 예: `?source=pr&repo=owner/repo&prNumber=123`

### 4. 글쓰기/수정 페이지 (/post/create/editor, /post/:id/edit)

#### 4.1 페이지 초기화

- 쿼리 파라미터를 확인하여 PR 기반 작성인지 일반 작성인지 구분한다
- PR 기반인 경우: 쿼리 파라미터의 PR 정보를 읽어온다
- 일반 작성인 경우: 빈 에디터 상태로 시작한다

#### 4.2 PR 정보 참조 (PR 기반인 경우)

- GitHub API를 통해 PR 본문과 커밋 로그를 가져온다
- 가져온 PR 정보를 에디터 하단 또는 사이드바에 참고용으로 표시한다
- 연결된 레포지토리와 PR 링크가 상단에 표시된다
- 사용자가 PR 정보를 참고하여 직접 글을 작성한다

#### 4.3 AI 생성 기능

- PR 정보가 있는 경우, "AI로 초안 생성" 버튼을 제공한다
- 버튼 클릭 시 AI API를 호출하여 PR 정보 기반 블로그 포스팅 초안을 생성한다
- 생성된 초안을 제목 입력창과 에디터 본문에 채운다
- AI 생성 후에도 자유롭게 수정 가능하다

#### 4.4 에디터 기능

- 마크다운 문법을 지원하는 에디터를 사용한다
- 제목 입력 필드가 있다

#### 4.5 저장 및 발행

- 발행 버튼을 누르면 포스팅이 로컬스토리지에 저장되고 상세 페이지로 이동한다
- 필수 입력값(제목, 본문) 검증을 수행한다
- PR 연동 정보도 함께 저장한다 (PR 기반인 경우)

### 5. GitHub API 연동

#### 5.1 서버 기반 API 연동

- GitHub API 요청은 Express 기반의 서버 환경에서 수행한다
- 프론트엔드는 Express 서버의 API를 통해 간접적으로 GitHub 데이터를 조회한다
- 토큰 정보는 서버에서만 관리하여 클라이언트에 노출되지 않도록 한다

#### 5.2 토큰 관리

- GitHub Personal Access Token (Classic)을 발급받는다
- `public_repo` 권한 사용
- 토큰은 `.env` 파일로 관리한다
- `.gitignore`에 `.env` 파일 추가 필수

### 6. 데이터 저장

#### 6.1 로컬스토리지 활용

- 포스팅 데이터를 로컬스토리지에 JSON 형태로 저장한다

- **Express** - API 서버
