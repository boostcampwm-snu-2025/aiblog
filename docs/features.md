# 📐 개발 기능 정리

## 필수 기능 (Feature)

### F1. 포스팅 관리

- **F1.1**: 포스팅 목록 조회 (메인 페이지 그리드)
- **F1.2**: 포스팅 상세 조회
- **F1.3**: 포스팅 생성 (일반/PR 기반)
- **F1.4**: 포스팅 수정
- **F1.5**: 포스팅 삭제

### F2. GitHub 연동

- **F2.1**: 레포지토리 검색
- **F2.2**: PR 목록 조회
- **F2.3**: PR 상세 정보 조회 (본문, 커밋 로그)
- **F2.4**: GitHub 토큰 인증

### F3. 마크다운 에디터

- **F3.1**: 마크다운 작성
- **F3.2**: 마크다운 렌더링
- **F3.3**: 제목 입력

### F4. AI 생성

- **F4.1**: "AI로 초안 생성" 버튼 제공
- **F4.2**: PR 정보 기반 콘텐츠 자동 생성
- **F4.3**: 생성된 콘텐츠 에디터 삽입
- **F4.4**: 프롬프트 최적화

### F5. 데이터 관리

- **F5.1**: 로컬스토리지 저장
- **F5.2**: 로컬스토리지 조회
- **F5.3**: 로컬스토리지 삭제

## 기능별 의존성

```
F2 (GitHub 연동)
  ↓
F4 (LLM API)
  ↓
F3 (마크다운 에디터)
  ↓
F5 (데이터 관리)
  ↓
F1 (포스팅 관리)
```

**설명**

1. GitHub 연동이 먼저 완성되어야 PR 정보를 가져올 수 있음
2. PR 정보로 LLM API를 호출하여 콘텐츠 생성
3. 생성된 콘텐츠를 에디터에서 수정
4. 완성된 콘텐츠를 저장
5. 저장된 콘텐츠를 조회/관리

## 데이터 흐름

### 1. PR 기반 포스팅 생성 플로우

```
사용자 입력 (repo 이름)
  → F2.1: 레포지토리 검색
  → F2.2: PR 목록 조회
  → 사용자 선택 (PR 선택)
  → F2.3: PR 상세 정보 조회
  → F4.1: LLM API 호출 (콘텐츠 생성)
  → F3.2: 에디터에 삽입
  → 사용자 수정 (F3.1)
  → F5.1: 로컬스토리지 저장
  → F1.2: 상세 페이지 표시
```

### 2. 일반 포스팅 생성 플로우

```
사용자 입력 (그냥 작성하기)
  → F3.1: 빈 에디터
  → 사용자 작성
  → F5.1: 로컬스토리지 저장
  → F1.2: 상세 페이지 표시
```

### 3. 포스팅 조회 플로우

```
F5.2: 로컬스토리지 조회
  → F1.1: 목록 표시
  → 사용자 선택
  → F1.2: 상세 페이지 표시
  → F3.2: 마크다운 렌더링
```
